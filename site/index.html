Tool-Wizard × Chainwright

Design your own tool belt.

One project, two faces. Chainwright is a growing library of recipe tools, each composed from a small set of recombinant primitives. Tool-Wizard is the conductor that fires up your whole coding tool belt — credentials, memory, sessions, GitHub state — on a specific project in seconds.

Composition here is shape-based: every primitive declares its slots, and every classifier returns the same envelope. Things connect because they agree on contracts — not because we've axiomatized the algebra. (No, it isn't a monad. Yet.)

Chainwright

The library/ consumer entry

Describe a friction in plain language. Chainwright classifies it to one of eight named patterns, then surfaces the recipe tools in the library that already use that pattern. You clone, remix, and learn the grammar by recognition — not by curriculum.

The eight friction patterns

  1. Memory Failure "I forgot what I captured before."
  2. State Tracking "I don't know what stage this is in."
  3. Repetitive Language "I keep retyping the same intro emails."
  4. Comparison Overload "I have 40 PDFs and can't rank them."
  5. Don't-Know-Next "What's my first move tomorrow?"
  6. Scattered Attention "Where is everyone on my team?"
  7. Collecting From Others "How do I gather input without chaos?"
  8. Time-Sensitive Triggers "I need reminders that branch on outcomes."

Named recipe shapes

Recipes cluster into named combos so the grammar is learnable: The Pipeline, The Pulse, The Drip, The Scorecard, The Ritual, The Reuse Kit. ~24 recipes drafted in the design library across these shapes.

Status. Diagnostic UI and recipe library are still in design. The classifier is currently keyword-based; LLM swap is a drop-in (preserves { pattern, confidence, evidence, alternatives, ranked_matches }).
Tool-Wizard

The conductor/ developer entry

Tool-Wizard fires up your whole coding tool belt on a specific project in one phrase. It runs four phases against the same repo, threading context from each into the next, and returns one dashboard so you know what's loaded and what needs attention before you start writing code.

The belt

  • Keysmith — credential health and injection (mcp__keysmith__*)
  • repo-memory — project canon, constraints, prior session state (mcp__repo-memory__*)
  • IGNITION — bounded coding sessions with checkpoints and receipts (mcp__ignition__*)
  • GitHub MCP — read/write bridge to the actual repository (mcp__github__*)

Three form factors

  • v1 — Cowork chat skill drafted
    "fire up Open-Case" / /tool-wizard → four-phase orchestration, one dashboard.
  • v2 — Terminal CLI planned
    tw fire ~/Open-Case from any terminal.
  • v3 — Mac Shortcut planned
    One menu-bar button that calls v2 on the foreground project.

Install

v1 is a skill bundle (a .skill file = zip with known extension) installed in Cowork. After packaging from skills/tool-wizard/SKILL.md, drop the file into Cowork — click Save skill — restart the chat. 

bash ~/My tool kit/Tool-Wizard/package.sh

Design principles, both faces

  • Each tool stays sovereign. Keysmith owns creds. IGNITION owns sessions. repo-memory owns project knowledge. GitHub MCP owns the repo. Tool-Wizard composes them — never duplicates.
  • Read first, write never (in v1). The conductor reads state and reports. Writes (init, begin a session, comment on a PR) happen only after the user explicitly asks.
  • Primitives stay recombinant. A primitive only belongs in the Chainwright library if at least one other recipe could plausibly use it. No domain-locked monoliths.
  • Confidence is marked explicitly. Slots and primitives carry Stable / Experimental / Custom labels. You always see what's proven vs. in-flight.
  • Plain text first. Memory, canon, constraints, sessions — all markdown / YAML on disk. No DB, no embeddings, no LLM calls inside the memory layer.
  • Don't call it monadic. Composition is contract-uniform. That's the property that makes monads composable — but the algebra (bind, return, laws) hasn't been lifted. Either spec it or describe it accurately.
